# Spec: 趋势列表固定 50 槽位与异常数字展示治理

## 目标 (Goals)
- 首页和单平台趋势列表统一按 `50` 个展示槽位渲染。
- 超过 `50` 条时仅展示前 `50` 条，不足 `50` 条时补空槽位，保持卡片长度稳定。
- 消除因描述缺失、异常热度字段、非连续 rank 导致的视觉错位和数字混淆。

## 非目标 (Non-goals)
- 不在本轮修改抓取器、数据库快照写入逻辑或第三方数据源本身。
- 不在本轮补充新的抓取平台。
- 不在本轮重做整套首页布局。

## 约束 (Constraints)
- 保持现有页面结构与交互，不引入新的 API 请求。
- 展示层允许对异常字段做兜底和归一化，但不改写数据库原始数据。
- 修改应兼容 `all` 视图和单平台视图。

## 范围 (Scope)
### In Scope
- `TrendList` 统一做 `50` 槽位截断/补齐。
- 列表行高统一，避免同榜单内高低不齐。
- 对明显为序号型而非热度型的 `hotValue` 隐藏展示。
- 时间线卡片中的 `count` 文案改为“采集记录”，降低与首页 50 条展示规则的混淆。

### Out of Scope
- 后端 rank 修复与历史快照清洗。
- 时间线统计口径变更。
- 新增自动化 E2E 用例。

## 验收标准 (Acceptance)
- 每个平台列表始终渲染 `50` 行展示槽位。
- 同一榜单内每一行高度一致，不因 `description` 有无而变化。
- `weixin` / `weixinvideo` 这类序号型 `hotValue` 不再显示右侧误导数字。
- 时间线中的大数值文案不再被理解为“首页应展示几百条”。

## 风险与回滚 (Risk / Rollback)
- 风险：前端展示 rank 改为槽位序号后，与上游原始 rank 不再一一对应。
- 风险：热度隐藏启发式若过宽，可能误伤真实热度字段。
- 回滚：本次改动仅涉及文档与前端组件，可按单提交回退。

## 实施清单 (3-7 条)
- [ ] 在 `TrendList` 内加入 `50` 槽位标准化逻辑。
- [ ] 为真实条目和空槽位统一列表行高度。
- [ ] 增加异常 `hotValue` 识别并在展示层隐藏。
- [ ] 将时间线 `count` 文案改为“采集记录”。
- [ ] 执行 `npm run lint` 与 `npx tsc --noEmit` 验证。
